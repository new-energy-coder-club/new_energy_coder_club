
# Tools

> 开发工具和自动化脚本目录

## 📋 目录说明

**用途**：提供开发和维护工具，支持自动化流程

**访问级别**：supporting

**最大深度**：2 层

**命名规范**：`tools/{tool-category}`

## 📁 子目录

- `automation/` - 自动化脚本和工具
- `testing/` - 测试工具和框架
- `deployment/` - 部署和发布工具
- `monitoring/` - 监控和分析工具
- `utilities/` - 通用工具和实用程序


## 📖 使用示例

- `tools/automation/ci-cd`
- `tools/testing/unit-tests`
- `tools/deployment/docker`


## 📝 文件要求

### 必需文件
- `README.md`

### 可选文件
- `INSTALL.md`
- `USAGE.md`


## 🚀 快速开始

1. 在相应子目录下创建项目文件夹
2. 添加项目 README.md 文件
3. 按照命名规范组织文件结构
4. 更新上级目录的索引文件

# 新目录结构设计文档

## 1. 设计原则

### 1.1 一次点击原则
- 核心项目最多2层目录即可访问
- 减少用户查找时间，提升开发体验

### 1.2 语义化命名
- 统一使用英文命名
- 采用kebab-case命名规范
- 避免特殊字符和空格

### 1.3 分类清晰
- 按功能和技术领域明确分类
- 建立清晰的项目生命周期管理

### 1.4 扩展性强
- 支持未来项目的灵活添加
- 标准化的项目结构模板

## 3. 目录功能说明

### 3.1 competitions/ - 竞赛项目
- **组织方式**: 按年份分类，每年下设具体竞赛类型
- **命名规范**: 使用英文简称，如robocon、smart-car等
- **访问路径**: competitions/2024/robocon/ (3层访问)

### 3.2 projects/ - 技术项目
- **组织方式**: 按技术领域分类
- **分类标准**:
  - ai/ - 人工智能相关项目
  - robotics/ - 机器人技术项目
  - embedded/ - 嵌入式系统项目
  - research/ - 科研横向项目
  - templates/ - 项目模板
- **访问路径**: projects/ai/energy-monitoring/ (3层访问)

### 3.3 shared/ - 共享资源
- **assets/**: 静态资源文件
  - images/ - 图片资源
  - videos/ - 视频资源
  - documents/ - 文档资源
- **models/**: 3D模型文件
- **libraries/**: 共享代码库
- **tools/**: 工具脚本

### 3.4 docs/ - 文档中心
- **index.md**: 文档首页
- **getting-started/**: 快速开始指南
- **development/**: 开发指南
- **deployment/**: 部署指南
- **api/**: API文档

### 3.5 scripts/ - 自动化脚本
- **setup.sh**: 环境搭建脚本
- **build.py**: 构建脚本
- **deploy.sh**: 部署脚本
- **migrate.py**: 迁移脚本

### 3.6 config/ - 配置文件
- **project.json**: 项目配置
- **navigation.yml**: 导航配置
- **templates.yml**: 模板配置

## 4. 标准化项目结构模板

每个项目目录应包含以下标准文件：

```
project-name/
├── README.md                    # 项目说明文档
├── CHANGELOG.md                 # 变更日志
├── docs/                        # 项目文档
│   ├── setup.md                 # 环境搭建
│   ├── usage.md                 # 使用指南
│   └── api.md                   # API文档
├── src/                         # 源代码
├── tests/                       # 测试代码
├── assets/                      # 项目资源
├── config/                      # 配置文件
└── scripts/                     # 项目脚本
```

## 5. 命名规范

### 5.1 目录命名
- 使用小写字母
- 单词间用连字符(-)分隔
- 避免使用下划线(_)和空格
- 不使用中文和特殊字符

**示例**:
- ✅ `energy-monitoring`
- ✅ `smart-car`
- ✅ `3d-printing`
- ❌ `energy_monitoring`
- ❌ `Smart Car`
- ❌ `节能减排`

### 5.2 文件命名
- README文件统一使用`README.md`
- 配置文件使用小写字母和连字符
- 文档文件使用描述性名称

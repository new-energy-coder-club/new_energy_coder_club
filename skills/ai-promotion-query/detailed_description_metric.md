# 获取歌曲宣推指标算子详细使用说明

## 能力概述

本算子用于**获取指定单首歌曲的宣推指标详情**，是宣推数据查询流程的**第二步**。通过本算子拉取单首歌在宣推链路上的详细表现数据（播放表现、完播、转化、投放节奏等），用于：

- 回答"这首歌宣推效果怎么样？"
- 回答"这首歌接下来该加投还是止损？"
- 作为后续**结合宣推知识库做指标分析**（见 SKILL.md 的"场景三：宣推指标分析"）的数据输入

> ⚠️ 本算子 **不硬编码 operatorCode**。调用前必须先通过 `search_apis` 定位真实 operatorCode，并用 `get_api_detail` 确认权威 schema。

---

## 调用入口

### Step 1 — 搜索算子

```
search_apis({ keyword: "获取歌曲宣推指标" })
```

从返回中定位名称/描述与"获取歌曲宣推指标"匹配的条目，记下 `name`（即 operatorCode）。

### Step 2 — 获取详情（强烈建议首次调用前执行）

```
get_api_detail({ name: "<上一步找到的 operatorCode>" })
```

重点阅读：

- `inputSchema`：确认 `songId` 是否必填、是否还需要时间窗口等参数
- `exampleInput` / `exampleOutput`：参数组装样例与返回结构参考
- `detailedDescription`：后端维护的最权威说明，**若与本文档冲突以后端为准**

### Step 3 — 调用

```
invoke_api({
  name: "<operatorCode>",
  arguments: {
    "songId": "<目标歌曲ID>"
    /* 其他字段以 get_api_detail 返回为准 */
  }
})
```

---

## 输入参数（参考骨架，以 `get_api_detail` 为准）

| 参数名      | 类型   | 必填  | 说明                                                            |
| ----------- | ------ | ----- | --------------------------------------------------------------- |
| `songId`    | string | ✅ 是 | 歌曲唯一标识，必须从"宣推概览"或上下文中已知的歌曲来源获取     |
| `startDate` | string | ❌ 否 | 统计起始日期（`yyyy-MM-dd`），若支持时间窗口则可传，默认以后端为准 |
| `endDate`   | string | ❌ 否 | 统计结束日期（`yyyy-MM-dd`），默认以后端为准                    |

### `songId` 约束

- **必填**；无 `songId` 时禁止直接调用
- 如果用户在问题里只给了歌名，优先先调用"宣推概览"定位到 `songId`，再来调用本算子
- 如果通过概览仍无法唯一定位歌曲，**不得**向用户追问 `songId`，明确返回无法定位、查询失败或系统繁忙时，才输出固定兜底文案：`抱歉哦，系统繁忙中暂时无法为您提供该歌曲宣推评估，您可以选择其他歌曲咨询或稍后再试一试～`

## 输出结果（参考骨架）

获取歌曲宣推指标返回通常包含以下几个维度（最终字段以 `get_api_detail` 的 `exampleOutput` 为准）：

### 1. 基础信息

| 字段名           | 类型    | 说明             |
| ---------------- | ------- | ---------------- |
| `songId`         | string  | 歌曲 ID          |
| `songName`       | string  | 歌曲名称         |
| `coverUrl`       | string  | 封面 URL         |
| `publishTimeStr` | string  | 发行/上架时间    |
| `lifecycleStage` | string  | 当前生命周期阶段 |

### 2. 核心宣推指标（具体字段以后端为准）

可能包含但不限于：

- 播放表现（如播放量、日均播放、趋势）
- 完播率
- 杠杆率 / 转化效率
- 投放节奏（投放天数、阶段、近期动作）
- 素材维度表现（如多版本素材 CTR）

> **对外规则提醒**：以上所有指标的**具体数值**仅作为 Agent 内部判断依据，**严禁**在对外回复中展示原始数字。转写为"表现较好 / 偏弱 / 一般 / 有提升空间 / 值得继续观察"等定性评价。

## 使用场景

| ---------------------------------------- | ---------------------------------------------- |
| "《XXX》这首歌宣推效果怎么样？"          | 输入对应`songId`，返回单首完整指标             |

---

## 与"宣推概览"的配合

本算子**只处理单首歌**。如果用户：

- 没有先查概览，且问的是单首歌并给了歌名 → 先调用"宣推概览"定位 `songId`，再调用本算子
- 问的是账号级或多首歌整体情况 → 不要循环调用本算子，直接用"宣推概览"回答
- 已在上下文里拿到 `songId`（如概览结果、做歌/发行结果透传） → 直接调用本算子

## 常见错误与处理

| 错误场景                | 可能原因                   | 处理建议                                              |
| ----------------------- | -------------------------- | ----------------------------------------------------- |
| `songId` 不存在         | 歌曲已删除 / 非当前账号    | 提示："歌曲不存在或无权访问"，并引导回到概览重选     |
| 歌曲不在宣推中          | 该歌未进入宣推链路         | 提示："该歌曲当前没有宣推数据"                       |
| `INVALID_ARGUMENT`      | `songId` 格式错误或字段缺失 | 回退到`get_api_detail` 重新确认 schema               |
| `UNAUTHORIZED`          | 登录态失效                 | 提示用户删除 `~/.tme-login/token.json` 后重跑（`tme-openapi` 自动重新登录）       |
| `UPSTREAM_ERROR` / 超时 | 后端异常                   | 重试 1-2 次，仍失败则按"系统繁忙"固定兜底文案输出     |

---

## 对外表达规则

本算子返回的**数字指标**对 Agent 而言是**内部依据**，不是对外直接输出。最终输出时：

- ❌ 禁止展示任何具体金额、投放金额、购买播放量、投放天数等数字
- ❌ 禁止展示播放量、完播率、杠杆率、转化效率等**原始指标数值**
- ✅ 只能给出定性评价和可落地的建议
- ✅ 建议必须基于真实查询数据，不得凭空生成

# 宣推概览算子详细使用说明

## 能力概述

本算子用于**获取当前登录用户的宣推歌曲概览**，属于腾讯音乐人宣推数据查询流程的**第一步入口**。通过本算子拉取用户名下所有处于宣推链路上的歌曲及其整体表现摘要，用于：

- 回答"我这些歌宣推情况如何？"这类账号级/歌曲集维度的问题
- 作为后续单首歌指标分析（见 `detailed_description_metric.md`）的候选池，从中选出目标 `songId`

> ⚠️ 本算子 **不硬编码 operatorCode**。在实际调用前，必须先通过 `search_apis` 搜索关键词（推荐："宣推概览"、"宣推歌曲"、"宣推列表"）定位真实的 operatorCode，再通过 `get_api_detail` 获取权威 `inputSchema` / `exampleOutput`，最终以后端返回为准。

---

## 调用入口

### Step 1 — 搜索算子

```
search_apis({ keyword: "宣推概览" })
```

如无命中，可换用以下备选关键词重试：

- "宣推歌曲"
- "宣推列表"
- "宣推数据"
- "歌曲宣推"

从返回中定位名称/描述与"宣推概览 / 宣推歌曲概览"匹配的条目，记下 `name`（即 operatorCode）。

### Step 2 — 获取详情（强烈建议首次调用前执行）

```
get_api_detail({ name: "<上一步找到的 operatorCode>" })
```

重点阅读：

- `inputSchema`：权威的入参结构（是否需要 `pageNo` / `pageSize` / 时间窗口等，以其为准）
- `exampleInput` / `exampleOutput`：参数组装样例
- `detailedDescription`：后端维护的最权威使用说明（若与本地文档冲突，**以后端为准**）

### Step 3 — 调用

```
invoke_api({
  name: "<operatorCode>",
  arguments: { /* 以 get_api_detail 返回的 inputSchema 为准 */ }
})
```

## 输入参数（参考骨架，以 `get_api_detail` 为准）

宣推概览通常是"查询当前登录用户的全部宣推歌曲"，当前用户信息通过登录态自动透传，不需要由调用方传入 `accountId` 等身份参数。

常见的可选参数（是否存在以后端 schema 为准）：

| 参数名     | 类型   | 必填  | 说明                                  |
| ---------- | ------ | ----- | ------------------------------------- |
| `pageNo`   | number | ❌ 否 | 分页页码，从 1 开始，默认 `1`         |
| `pageSize` | number | ❌ 否 | 每页条数，默认 `10`，上限以后端为准   |

> 若 `get_api_detail` 返回的 `required` 中出现本表未列出的字段，请完全以后端为准，不要照搬本文档。

---

## 输出结果（参考骨架）

宣推概览接口的返回通常包含两类信息：

1. **账号级概览**：如是否曾经有过宣推歌曲、当前有多少首歌在宣推、整体的表现标签等
2. **单曲摘要列表**：每首参与宣推的歌曲的基础字段 + 宣推表现摘要

### 单曲摘要字段（常见字段，以后端返回为准）

| 字段名            | 类型    | 说明                                                  |
| ----------------- | ------- | ----------------------------------------------------- |
| `songId`          | string  | 歌曲唯一标识（**后续查询单首指标时必须使用此 ID**）   |
| `songName`        | string  | 歌曲名称                                              |
| `coverUrl`        | string  | 封面图 URL                                            |
| `duration`        | integer | 歌曲时长（秒）                                        |
| `publishTimeStr`  | string  | 发行/上架时间                                         |
| `promotionStatus` | string  | 宣推状态（如"投放中"/"已结束"等，以后端枚举为准）     |
| `performanceTag`  | string  | 表现标签（如"表现较好"/"一般"/"有提升空间"等定性标签）|

> 具体的字段命名、类型和枚举值**完全以 `get_api_detail` 返回的 schema 和 `detailedDescription` 为准**，本文档仅作为参考骨架。

## 使用场景

| 用户问法                           | 本算子的作用                                              |
| ---------------------------------- | --------------------------------------------------------- |
| "我这些歌宣推情况怎么样？"         | 直接调用，返回账号级概览 + 单曲摘要列表                   |
| "我名下有哪些歌在投放？"           | 直接调用，从返回里筛选`promotionStatus=投放中` 的条目     |
| 单首歌问题但歌名模糊，需先定位歌曲 | 先调用概览取候选池，从中匹配用户所说的歌名得到`songId`    |

---

## 与"获取歌曲宣推指标"算子的配合

宣推概览只提供**摘要级**信息，不包含单首歌的完整指标明细。若用户进一步追问某首歌的宣推诊断、投放表现、优化建议，应当：

1. 从本算子返回里拿到目标歌曲的 `songId`
2. 切换到"获取歌曲宣推指标"算子（见 `detailed_description_metric.md`）继续查询

---

## 常见错误与处理

| 错误场景                | 可能原因         | 处理建议                                              |
| ----------------------- | ---------------- | ----------------------------------------------------- |
| 返回列表为空            | 当前账号无宣推歌曲 | 提示用户："您当前没有在宣推的歌曲"                  |
| `UNAUTHORIZED`          | 登录态失效       | 提示用户重新执行`tme-login` Skill                     |
| `INVALID_ARGUMENT`      | 参数不符合 schema | 回退到`get_api_detail` 重新确认 schema，修正参数再试 |
| 请求超时 /`UPSTREAM_ERROR` | 后端响应异常     | 重试 1-2 次，仍失败则提示："网络异常，请稍后再试"     |

---

## 对外表达规则（必须遵守 `ai-promotion/readme.md`）

本算子返回的数据是 Agent 的**内部依据**，不是直接对外的展示内容。：

- ❌ 禁止向用户展示任何具体金额、投放金额、购买播放量、投放天数等数字
- ✅ 只能转写为"表现较好 / 偏弱 / 一般 / 有提升空间 / 值得继续观察"等**定性评价**
- ❌ 明确返回无法定位、查询失败或系统繁忙时，才输出固定兜底文案：`抱歉哦，系统繁忙中暂时无法为您提供该歌曲宣推评估，您可以选择其他歌曲咨询或稍后再试一试～`
---

## 后续操作

从概览结果中选出目标歌曲后：

1. 记录该歌曲的 `songId` 和 `songName`
2. 切换到"获取歌曲宣推指标"算子进行详细指标查询（见 `detailed_description_metric.md`）
3. 结合宣推知识库（后续补充）生成诊断与建议

# Consulting Analysis

面向业务/产品/技术决策场景的结构化咨询分析方法。目标是把模糊问题变成可执行结论，并给出可落地的行动建议与风险边界。

## 何时调用

- 用户说“帮我分析/诊断/给建议/做方案对比/写一份报告/做决策”
- 需求不清晰，需要澄清目标、约束、利益相关方与评估指标
- 需要把信息整理为：结论、证据、权衡、行动计划

## 输出结构（默认）

- 结论：1-3 条可直接执行的建议
- 目标与范围：要解决什么、不解决什么
- 现状与问题：关键事实、痛点、根因假设
- 选项与权衡：备选方案、成本/收益、风险/依赖
- 推荐方案：为什么选它、成功标准、里程碑
- 风险与对策：主要风险、缓解手段、监控指标
- 下一步：最小可行行动清单（按优先级）

## 常用框架（按需选用）

- MECE 拆解：维度完整且不重叠
- 2x2 决策：收益/成本、影响/可行性、风险/回报
- 假设驱动：列假设→验证路径→最小验证实验
- 根因分析：5 Whys、鱼骨图（文字化）
- 商业模型：价值主张/用户/渠道/成本/收入

## 交互规则

- 先给“可用的默认假设”，并标注哪些假设可被替换
- 信息不足时，最多提出 3 个高价值问题，其余用假设继续推进
- 用清晰的指标定义“成功”，避免抽象词

## 示例提示

- “我们有三种上线方案，帮我用 2x2（影响/可行性）做对比并给推荐。”
- “把这个产品问题用 MECE 拆解，给出根因假设、验证方式和下一步行动清单。”
